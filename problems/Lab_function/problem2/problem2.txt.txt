2. การผ่านไปยังกระบวนความชนิด Call-by-reference
--------------------------------------------------------- 
ในการเขียนโปรแกรมเพื่อรับวันที่จากแป้นพิมพ์เก็บในตัวแปร ผู้เขียนโปรแกรมต้องตรวจสอบว่า
ผู้ใช้ป้อนวันที่สมเหตุสมผล (valid) หรือไม่ กล่าวคือช่วงของวันที่ที่เป็นไปได้ต้องอยู่ระหว่าง 1 - 31
หากข้อมูลที่ผู้ใช้ป้อนไม่อยู่ในพิสัยดังกล่าวนี้ แสดงว่าข้อมูลไม่อยู่ในย่านที่ถูกต้องให้แสดงข้อความ
เตือนผู้ใช้ และให้ผู้ใช้ป้อนวันที่ใหม่จนกว่าจะถูกต้อง
     เพื่อให้การดำเนินการสะดวก และสามารถนำไปดัดแปลงใช้กับโปรแกรมอื่นๆที่มีการทำงานใน
ลักษณะเดียวกันได้ ผู้เขียนโปรแกรมจึงตัดสินใจกำหนดให้การรับวันที่และการตวรจสอบดังกล่าว
เป็นโปรแกรมย่อยชื่อ getdate()  
     เมื่อทำการ compile และให้โปรแกรมทำงาน และทดลองป้อนวันที่เป็น -1, 0, 32, และ 33 โปรแกรม
จะต้องปรากฏข้อความเตือนและให้ผู้ใช้ป้อนวันที่ใหม่ได้อย่างถูกต้อง เมื่อผู้ใช้ป้อนวันที่ในช่วง 1- 31
โปรแกรมย่อยจะเลิกทำงาน แล้วโปรแกรมหลักจะพิมพ์ค่าของวันที่ออกทางหน้าจอ 

กำหนดรูปแบบของโปรแกรมย่อยและโปรแกรมหลักดังนี้
#include   <stdio.h>

/* รูปแบบการใช้งานของโปรแกรมย่อย */
void   getdate ( int   date );

/* M A I N   P R O G R A M */
int   main ( )
{
     int        dd = 0;

     /* เรียกใช้งานโปรแกรมย่อยที่เขียนขึ้นเพื่อรับค่าวันที่เก็บในตัวแปร dd และทำการพิมพ์ค่าที่ได้ */
     getdate(dd);	
     printf("date = %d\n", dd);

     return 0;
}

คำแนะนำ
------------
เนื่องจากการรับวันที่ต้องรับอย่างน้อยหนึ่งครั้ง - ไม่รับไม่ได้
จึงควรเลือกใช้โครงสร้างทำซ้ำแบบ do..while ซึ่งเป็น one-or-more เพื่อให้สอดคล้องกับการทำงานจริง
